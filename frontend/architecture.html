2 Multi-Agent Orchestration

The core intelligence is a LangGraph state machine with 5 specialized agents working in concert.

  1. Semantic Cache Agent
    Embeds every incoming query using all-MiniLM-L6-v2 and compares against stored entries via cosine similarity. If similarity ≥ 0.88, the cached answer is returned immediately — zero LLM calls.
  2. Query Planner Agent
    An LLM analyzes the question and outputs a structured plan: sub-questions (for complex/decomposable queries), retrieval strategy (hybrid, hybrid+web, web, or vector), whether code execution is needed, and complexity level. Follow-up questions are rewritten to be self-contained using conversation history.
  3. Retrieval Agent
    Executes the planned strategy. Supports 4 retrieval methods: Vector search (ChromaDB dense embeddings), BM25 (sparse keyword), Hybrid (Reciprocal Rank Fusion of both), and Web search (Tavily). All results are merged and reranked by a cross-encoder/ms-marco-MiniLM-L-6-v2 model for maximum precision.
  4. Critic / Validation Agent
    An LLM scores the retrieved context from 0.0–1.0 for sufficiency. If the score falls below the threshold (default 0.4), the query is reformulated and re-retrieved (up to 3 attempts). This autonomous critique loop dramatically improves answer quality.
  5. Synthesis Agent
    Generates the final answer with inline source citations [1], [2], adapting the response style to the query complexity (concise for simple, structured sections for complex). Includes a groundedness verification step that checks every claim against sources.

3 Technology Stack

Every component was chosen for a specific reason — balancing performance, cost, and maintainability.

Backend Framework FastAPI + Uvicorn Async-native, automatic OpenAPI docs, high throughput, easy deployment behind Nginx.
Agent Orchestration LangGraph State-graph architecture gives explicit control over the multi-agent pipeline — far more debuggable than linear chains.
LLM Groq (Llama 3.3 70B) 2000+ tokens/second inference, generous free tier. Drop-in replaceable via config.

Keyword Search rank-bm25 BM25 Okapi — the standard for lexical search. Complements dense embeddings perfectly.

Web Search Tavily Purpose-built for LLM agents. Returns clean, structured content.
MCP Protocol mcp SDK (Anthropic) Industry standard for LLM tool access. Exposes 7 tools to any MCP-aware client.
Document Parsing pypdf + python-docx + EasyOCR PDF, DOCX, TXT support with OCR fallback for scanned documents.
Containerization Docker Reproducible builds, easy CI/CD, one-command deployment.
Cloud AWS EC2 (Ubuntu 22.04) Cost-effective single-instance deployment. t3.medium is sufficient.
Reverse Proxy Nginx TLS termination, static file serving, rate limiting, security headers.
Security fail2ban Brute-force protection for SSH. Bans IPs after repeated failed attempts.

4 Docker Deployment

The application is fully containerized for reproducible builds and easy deployment.

💻

Multi-stage Ready

The Dockerfile uses a single python:3.13-slim stage. System dependencies for EasyOCR, pdf2image, and OpenCV are installed in under 100 MB.

🔒

Health Check

Docker HEALTHCHECK pings /health every 30s. AWS ECS uses this for automatic container replacement.

📈

Persistent Volume

The chroma_db/ vector store directory is mounted as a Docker volume so embeddings survive container restarts. 

# Build the image
docker build -t rag-chatbot .

# Run locally (with API keys)
docker run -p 8000:8000 -p 8100:8100 \
  -e GROQ_API_KEY=gsk_your_key \
  -e TAVILY_API_KEY=tvly_your_key \
  -v chroma_data:/app/chroma_db \
  rag-chatbot

# Tag and push to ECR
aws ecr create-repository --repository-name rag-chatbot
docker tag rag-chatbot:latest \
  <account>.dkr.ecr.<region>.amazonaws.com/rag-chatbot:latest
docker push <account>.dkr.ecr.<region>.amazonaws.com/rag-chatbot:latest

5 AWS EC2 — Nginx — fail2ban

Production-ready deployment with security hardening on a single Ubuntu 22.04 instance.

🏛

EC2 Setup

Instance type: t3.medium (2 vCPU, 4 GB RAM). AMI: Ubuntu 22.04 LTS. Security group: ports 22 (SSH), 80 (HTTP), 443 (HTTPS) from 0.0.0.0/0. An EFS volume is mounted at /mnt/efs for ChromaDB persistence.

🛰

Nginx Configuration

Nginx acts as a reverse proxy, forwarding requests to Uvicorn on port 8000. It handles SSL termination (via Let's Encrypt), static file caching, and adds security headers (HSTS, X-Frame-Options, etc.).

🛡

fail2ban Security

fail2ban monitors SSH authentication logs. After 5 failed login attempts within 10 minutes, the offending IP is banned for 24 hours. This protects against brute-force attacks on the EC2 instance. 

# ─── EC2 User Data Script ───
#!/bin/bash
apt-get update
apt-get install -y docker.io nginx fail2ban

# Start Docker

systemctl start docker

# Pull & run the container
aws ecr get-login-password --region us-east-1 | \
  docker login --username AWS --password-stdin \
  <account>.dkr.ecr.us-east-1.amazonaws.com

docker run -d -p 127.0.0.1:8000:8000 \

  -e TAVILY_API_KEY="..." \

# ─── Nginx Config (/etc/nginx/sites-available/rag) ───
server {
    listen 80;

7 Key Design Decisions

Trade-offs and rationale behind the major architectural choices.

Why LangGraph over LangChain's LCEL?

LCEL (LangChain Expression Language) is great for linear chains but struggles with branching, looping, and state management. LangGraph's explicit node/edge model maps directly to the critique-retry loop, making it far easier to debug and extend.

Why ChromaDB over Pinecone/Weaviate?

Zero external dependencies. ChromaDB is an embedded database that persists to disk — no cloud service, no API keys, no network latency. Perfect for a self-contained deployment on a single EC2 instance.

Why BM25 in addition to embeddings?

Dense embeddings (sentence-transformers) excel at semantic search but can miss exact keyword matches — especially for technical terms, code snippets, and proper names. BM25 catches these cases. The hybrid approach with RRF fusion gives the best of both worlds.

Why a cross-encoder reranker?

Bi-encoder embeddings (like all-MiniLM-L6-v2) are fast but approximate. A cross-encoder scores each (query, document) pair jointly, producing much more accurate relevance judgments. The reranker only runs on the top-K results, so the latency impact is minimal.

Why Nginx in front of Uvicorn?

While Uvicorn is a competent HTTP server, Nginx provides SSL termination, static file serving with caching, request buffering, rate limiting, and security headers — all handled by a battle-tested web server. This also makes it trivial to add a second Uvicorn worker for horizontal scaling.

Why fail2ban?

EC2 instances are constantly probed by automated SSH brute-force attacks. fail2ban monitors auth logs and dynamically blocks offending IPs via iptables. It's lightweight, zero-maintenance, and highly effective.
